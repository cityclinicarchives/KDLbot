Версия под один price_list.xlsx

Что изменено:
1. complex_price_list.xlsx больше не используется.
2. Все комплексы перенесены в price_list.xlsx.
3. pricing.py читает только price_list.xlsx.
4. Комплекс определяется по заполненному столбцу components.
5. Поиск Python работает по единому списку: одиночные анализы + комплексы.
6. alias_groups.xlsx остается отдельным словарем соответствий.

Важно:
- Не загружайте complex_price_list.xlsx в Railway для этой версии.
- Если добавляете новый комплекс, добавляйте его строкой в price_list.xlsx и заполняйте components.
