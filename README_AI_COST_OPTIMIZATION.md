# Оптимизация расходов на ИИ

В этой версии ИИ используется только там, где он действительно нужен:

1. Текстовые сообщения пациента не отправляются в OpenAI. Они сразу передаются в `pricing.py`.
2. Голосовые сообщения проходят только через `gpt-4o-mini-transcribe`; второй текстовый GPT-запрос после транскрибации отключен.
3. Изображения сначала пробуют локальный OCR, если он доступен в окружении. Если OCR не справился, используется vision-модель.
4. Перед отправкой в vision изображение уменьшается до 1280 px по длинной стороне и сжимается в JPEG quality=75.
5. Для vision установлен `detail=low` и `max_output_tokens=300`.
6. В лабораторный чат по-прежнему отправляется информация о токенах, если использовался OpenAI.

Рекомендуемые переменные Railway:

```env
OPENAI_TEXT_MODEL=gpt-4.1-nano
OPENAI_VISION_MODEL=gpt-4.1-nano
OPENAI_AUDIO_MODEL=gpt-4o-mini-transcribe
```

`OPENAI_TEXT_MODEL` оставлен как fallback для будущих сценариев, но в обычном тексте и после голоса сейчас не используется.
